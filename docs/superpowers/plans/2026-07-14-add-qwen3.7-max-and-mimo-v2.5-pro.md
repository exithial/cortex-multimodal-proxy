# Add `proxy/qwen3.7-max` and `proxy/mimo-v2.5-pro` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Register two new text-only OpenCode Go models (`qwen3.7-max` and `mimo-v2.5-pro`) as proxy brains so OpenCode and Claude Code clients can route requests to them, while preserving image vision via the existing MiMo V2.5 senses layer.

**Architecture:** Append two entries to `BRAIN_MODELS` in `src/services/brainRegistry.ts`. The existing `multimodalProcessor` already keys vision routing off the `proxy/` prefix and `mimoSensesService.isAvailable()`, so image vision is enabled automatically. `qwen3.7-max` uses the Anthropic-format endpoint at `/v1/messages` (already supported by `endpoint: "anthropic"` in `BrainModelEntry` and `openAIToAnthropicPayload` in `opencodeGoService.ts:25-118`); `mimo-v2.5-pro` uses the OpenAI-format endpoint. The existing `mimo-v2.5` passthrough remains unchanged (coexistence).

**Tech Stack:** TypeScript 5, Node.js >= 20.x, Vitest, ESLint, OpenCode Go API (Anthropic + OpenAI compatible formats), MiMo V2.5 senses layer, Gemini fallback.

## Global Constraints

- Node.js >= 20.x (per `CLAUDE.md` § Environment).
- Build must pass (`npm run build`).
- All unit tests must pass (`npm run test:unit`).
- Lint clean (`npm run lint`).
- Use exact string matches for model routing, not `includes`/prototype checks.
- Use `Object.hasOwn()` for registry checks (prototype safety).
- No new dependencies.
- No changes to `opencodeGoService`, `multimodalProcessor`, `mimoSensesService`, `geminiService`, `anthropicAdapter`, `index.ts`, or `.env.example`.
- No new Claude Code aliases.
- `mimo-v2.5` passthrough is preserved (coexistence with new `proxy/mimo-v2.5-pro` brain).
- All four brains must remain text-only (`multimodal: false`) and use `thinking: true`.
