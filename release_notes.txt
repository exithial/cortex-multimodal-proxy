Cortex Multimodal Proxy v3.0.0 — "Cortex Sensorial v3"

The 3.0.0 release is a complete rebuild of the proxy around the OpenCode Go subscription and a new lightweight image-description pipeline powered by MiMo V 2.5. The proxy keeps full compatibility with the two clients it was built for (OpenCode CLI and Claude Code), while running leaner and cheaper on upstream API costs.

What changed for users

- The project is now named Cortex Multimodal Proxy. The GitHub repository has been renamed to exithial/cortex-multimodal-proxy.
- Claude Code users get the same haiku / sonnet / opus experience. Internally, haiku is now routed through the multimodal MiMo V 2.5 path (no more Gemini detour for fast or short prompts).
- OpenCode users see a curated 3-model list instead of the previous 5-variants DeepSeek list. Pick the brain that matches the task; the proxy handles the rest.
- Images attached to text-only requests are now described by MiMo V 2.5 instead of going through Gemini. For end users this means lower per-image cost and faster responses for everyday questions.
- Audio, video, and large PDFs continue to use Gemini 2.5 Flash when configured. The Gemini integration is now optional and only engaged for non-image media, so you can run the proxy without a Gemini key for text, code, and image workflows.
- All brains run with maximum-quality reasoning enabled by default. Expect deeper reasoning in long-horizon tasks.
- A single OpenCode Go subscription replaces separate DeepSeek and Google API bills. One key, one invoice.

What changed for operators

- The model registry is the new single source of truth. Adding or swapping a brain is a one-line change.
- Upstream 503 / 502 / 429 responses trigger an automatic retry with exponential backoff, honoring the Retry-After header. Short blips no longer surface as user-visible failures.
- The container image and Docker Compose service have been renamed to cortex-multimodal-proxy.

Upgrade notes

- The env file now expects OPENCODE_GO_API_KEY (from opencode.ai/auth). DEEPSEEK_API_KEY and DEEPSEEK_BASE_URL are no longer read.
- GEMINI_API_KEY is optional. Leave it unset if you do not use audio, video, or large PDFs.
- Existing Claude Code setups work unchanged; the alias mapping is now controlled via CLAUDE_HAIKU_MODEL, CLAUDE_SONNET_MODEL, and CLAUDE_OPUS_MODEL environment variables.
- Existing OpenCode setups should update the model references to the proxy/glm-5.2 and proxy/deepseek-v4-pro identifiers (vision-direct is no longer a model entry).
